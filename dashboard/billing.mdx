## Managing Subscription

### Upgrading Your Plan

<Steps>
  <Step title="Navigate to Billing">
    Go to Dashboard → Settings → Billing
  </Step>

  <Step title="Click 'Upgrade Plan'">
    Review available plans and features
  </Step>

  <Step title="Select New Plan">
    ```yaml
    Current Plan: Starter ($29/month)
    New Plan: Pro ($99/month)

    Changes:
    ✓ Validations: 10,000 → 100,000/month
    ✓ Projects: 3 → 10
    ✓ Team Members: 3 → 10
    ✓ Data Retention: 30 → 90 days
    ✓ Advanced Analytics
    ```
  </Step>

  <Step title="Confirm Upgrade">
    Review prorated charges and confirm

    ```
    Pro Plan: $99/month
    Prorated credit from Starter: -$15
    Total due today: $84

    Next billing date: Feb 1, 2024
    ```
  </Step>

  <Step title="Access New Features">
    Upgrade takes effect immediately

    New limits and features are available
  </Step>
</Steps>

<Info>
  Upgrades are prorated - you only pay for the time remaining in your billing cycle.
</Info>

### Downgrading Your Plan

<Steps>
  <Step title="Navigate to Billing">
    Go to Dashboard → Settings → Billing
  </Step>

  <Step title="Click 'Change Plan'">
    Select lower-tier plan
  </Step>

  <Step title="Review Impact">
    ```yaml
    Current Plan: Pro ($99/month)
    New Plan: Starter ($29/month)

    Impact:
    ⚠ Validations: 100,000 → 10,000/month
    ⚠ Projects: 10 → 3 (archive 7 projects)
    ⚠ Team Members: 10 → 3 (remove 7 members)
    ⚠ Data Retention: 90 → 30 days

    Action Required:
    - Archive or delete 7 projects
    - Remove 7 team members
    - Data older than 30 days will be deleted
    ```
  </Step>

  <Step title="Take Required Actions">
    Complete required actions before downgrade:
    - Archive excess projects
    - Remove team members
    - Export data if needed
  </Step>

  <Step title="Confirm Downgrade">
    Downgrade takes effect at end of billing cycle

    ```
    Current Period: Jan 1 - Jan 31
    Downgrade effective: Feb 1, 2024
    Continued access until: Jan 31, 2024
    ```
  </Step>
</Steps>

<Warning>
  Downgrades take effect at the end of your billing cycle. You keep full access until then.
</Warning>

### Reducing Costs

<AccordionGroup>
  <Accordion title="Use Test Keys for Development" icon="flask">
    ```typescript
    // Test keys don't count toward quota
    const apiKey = process.env.NODE_ENV === 'production'
      ? process.env.VOUCH_LIVE_KEY    // Counts toward quota
      : process.env.VOUCH_TEST_KEY;   // Free unlimited

    // Potential savings: $29-99/month
    ```
  </Accordion>

  <Accordion title="Cache Validation Results" icon="database">
    ```typescript
    // Cache for 24 hours to reduce API calls
    const cached = cache.get(email);
    if (cached) return cached;

    const result = await vouch.validate(email);
    cache.set(email, result, 86400); // 24 hours

    // Potential savings: 30-50% reduction in API calls
    ```
  </Accordion>

  <Accordion title="Client-Side Pre-Validation" icon="filter">
    ```typescript
    // Basic validation before API call
    if (!isBasicEmailValid(email)) {
      return { valid: false };
    }

    // Only call API for emails that pass basic checks
    const result = await vouch.validate(email);

    // Potential savings: 10-20% reduction in API calls
    ```
  </Accordion>

  <Accordion title="Batch Processing" icon="layer-group">
    ```typescript
    // Process in batches during off-peak hours
    const batch = await getBatchEmails();

    for (const email of batch) {
      await vouch.validate(email);
      await sleep(100); // Rate limit friendly
    }

    // Spread usage over time to avoid spikes
    ```
  </Accordion>

## Failed Payments

### Payment Failure Process

If a payment fails:

<Steps>
  <Step title="Immediate Notification">
    Email sent to account owner

    ```
    Subject: Payment Failed - Action Required

    Your payment of $99 failed on Jan 1, 2024.

    Reason: Card declined
    Card: Visa •••• 4242

    Please update your payment method to avoid service interruption.
    ```
  </Step>

  <Step title="Retry Schedule">
    ```yaml
    Attempt 1: Immediate
    Attempt 2: 3 days later
    Attempt 3: 5 days later
    Attempt 4: 7 days later
    ```
  </Step>

  <Step title="Grace Period">
    Service continues for 7 days

    ```
    Days Remaining: 7
    Status: Payment Past Due
    Action Required: Update payment method
    ```
  </Step>

  <Step title="Service Suspension">
    After 7 days, service is suspended

    ```
    Status: Suspended
    Reason: Payment failed
    Action: Update payment method to reactivate
    ```
  </Step>

  <Step title="Reactivation">
    Update payment method to reactivate

    Service resumes immediately upon payment
  </Step>
</Steps>

### Updating Payment After Failure

```yaml
Steps to Reactivate:
1. Login to dashboard
2. Go to Billing → Payment Methods
3. Update or add new payment method
4. Click "Retry Payment"
5. Service reactivates immediately
```
