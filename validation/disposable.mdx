## Overview

Disposable email detection identifies temporary email services that users employ to avoid providing their real email address. These services offer short-lived inboxes that require no registration and are commonly used for spam, fraud, or avoiding long-term commitment.

<Warning>
  Disposable emails are a major indicator of low-quality signups and should typically be blocked.
</Warning>

## What Are Disposable Emails?

Disposable email services (also called temporary email, throwaway email, or DEA) provide instant, temporary email addresses that expire after a set time period.

## How Detection Works

### Database Lookup

Vouch maintains a comprehensive database of 5,000+ known disposable email domains:

```typescript
// Simplified detection logic
async function isDisposable(email: string): Promise<boolean> {
  const domain = email.split('@')[1].toLowerCase();

  // Check against database
  const isInDatabase = await disposableDomains.includes(domain);

  // Check against patterns
  const matchesPattern = /^(temp|disposable|throwaway|fake)mail/i.test(domain);

  return isInDatabase || matchesPattern;
}
```

### Pattern Recognition

Beyond the database, Vouch uses pattern matching to detect new or unknown services:

- Domains containing "temp", "disposable", "trash", "throw"
- Randomly generated domain names
- Suspicious TLDs (.tk, .ml, .ga)
- Domain age and registration patterns

### Database Updates

The disposable email database is updated regularly:

- **Daily updates** from community sources
- **Automatic detection** of new services
- **Manual curation** of reported domains
- **False positive removal** when legitimate services are flagged

## Configuration Options

### Recommended Setting

<Card title="BLOCK" icon="ban" color="#ef4444">
  For most applications, disposable emails should be blocked. They indicate users who don't want to provide real contact information.
</Card>

### Alternative Settings

<Tabs>
  <Tab title="BLOCK (Strict)">
    **Best for:** SaaS, marketplaces, social networks

    Reject all disposable emails immediately. Highest security, may frustrate privacy-conscious users.

    ```json
    {
      "disposable": {
        "action": "block",
        "allowPrivacyServices": false
      }
    }
    ```
  </Tab>

  <Tab title="FLAG (Moderate)">
    **Best for:** Content sites, newsletters, low-risk signups

    Allow signup but mark for review or reduced privileges.

    ```json
    {
      "disposable": {
        "action": "flag",
        "allowPrivacyServices": true
      }
    }
    ```
  </Tab>

  <Tab title="ALLOW (Permissive)">
    **Best for:** Developer tools, testing services

    Track disposable emails but don't restrict them.

    ```json
    {
      "disposable": {
        "action": "allow",
        "trackOnly": true
      }
    }
    ```
  </Tab>
</Tabs>

## User Experience

### Handling Blocked Disposable Emails

Provide clear, helpful error messages:

<CodeGroup>
```jsx Good UX
{result.success && result.data && !result.data.checks.disposable?.pass && (
  <Alert variant="error">
    <p>Please use a permanent email address.</p>
    <p className="text-sm">
      Temporary email services are not allowed.
      Use your personal or work email instead.
    </p>
  </Alert>
)}
```

```jsx Avoid This
{result.success && result.data && !result.data.checks.disposable?.pass && (
  <p>Error: Disposable email detected (
    {result.data.checks.disposable.metadata?.domain}
  )</p>
)}
```
</CodeGroup>

### Alternative Solutions

Offer alternatives for privacy-conscious users:

```jsx
{result.success && result.data && !result.data.checks.disposable?.pass && (
  <Alert variant="warning">
    <p>Temporary email detected. We need a permanent address to:</p>
    <ul>
      <li>Send account recovery information</li>
      <li>Provide important updates</li>
      <li>Verify your identity</li>
    </ul>
    <p className="text-sm">
      Privacy tip: Create a dedicated email for services like ours,
      or use privacy services like Apple Hide My Email or Firefox Relay.
    </p>
  </Alert>
)}
```

## API Integration

### Checking Only Disposable

Vouch automatically runs all enabled checks. You can check specific validation results:

```javascript
const result = await vouch.validate('user@example.com');

if (result.success && result.data) {
  const { checks } = result.data;

  if (!checks.disposable?.pass) {
    console.log('Disposable email detected!');
    console.log('Domain:', checks.disposable.metadata?.domain);
    console.log('Service:', checks.disposable.metadata?.service);
  } else {
    console.log('Email is not disposable');
  }
}
```
