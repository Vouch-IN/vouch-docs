## Overview

Device fingerprinting creates a unique identifier for each device accessing your application, allowing you to track the same device across multiple signups, detect multi-accounting, and identify sophisticated fraud patterns.

<Note>
  Vouch uses **FingerprintJS Pro** technology for 99.5% device identification accuracy. Fingerprints are automatically collected by client SDKs.
</Note>

## How It Works

Device fingerprinting analyzes hundreds of browser and device signals to create a unique identifier:

## The Fingerprinting Process

<Steps>
  <Step title="Signal Collection">
    Client SDK collects 100+ browser and device signals
  </Step>
  <Step title="Fingerprint Generation">
    Signals are hashed into a unique device identifier
  </Step>
  <Step title="Server Validation">
    Fingerprint sent with validation request
  </Step>
  <Step title="History Check">
    Check if device has validated other emails
  </Step>
  <Step title="Risk Assessment">
    Analyze patterns: velocity, diversity, suspicious behavior
  </Step>
  <Step title="Action Recommendation">
    Return allow, flag, or block based on device history
  </Step>
</Steps>

Device fingerprints are automatically collected by Vouch client SDKs:

  <Tab title="JavaScript">

## Risk Signals

### Email Velocity

How many emails validated from this device over time:

| Velocity | Previous Emails | Timeframe | Risk Level |
|----------|----------------|-----------|------------|
| Very Low | 0-1 | Any | Low |
| Low | 2-5 | > 1 week | Low |
| Medium | 6-10 | > 1 week | Medium |
| High | 11-20 | < 1 week | High |
| Very High | 20+ | < 1 day | Critical |

### Risk Factors

<AccordionGroup>
  <Accordion title="RAPID_ACCOUNT_CREATION" icon="gauge-high">
    Multiple emails validated in a short time period.

    **Threshold:** 5+ emails in 1 hour
    **Indicates:** Automated bot, referral abuse, multi-accounting
  </Accordion>

  <Accordion title="HIGH_VOLUME_DEVICE" icon="chart-line">
    Device has validated many emails total.

    **Threshold:** 20+ total emails
    **Indicates:** Account farming, professional fraudster
  </Accordion>

  <Accordion title="SUSPICIOUS_VELOCITY" icon="clock">
    Unusual pattern of account creation.

    **Example:** 10 accounts in 5 minutes, then stops
    **Indicates:** Scripted automation, trial abuse
  </Accordion>

  <Accordion title="INCOGNITO_MODE" icon="eye-slash">
    User in private/incognito browsing mode.

    **Risk:** Medium (can be legitimate privacy concern)
    **Indicates:** May want to hide identity or avoid tracking
  </Accordion>

  <Accordion title="BOT_LIKELY" icon="robot">
    Device exhibits bot-like characteristics.

    **Signals:** No mouse movement, perfect timing, missing sensors
    **Indicates:** Automated script, headless browser
  </Accordion>

  <Accordion title="TAMPERED_FINGERPRINT" icon="screwdriver-wrench">
    Signs of fingerprint spoofing or manipulation.

    **Signals:** Inconsistent signals, known spoofing tools
    **Indicates:** Sophisticated fraud attempt
  </Accordion>
</AccordionGroup>

## Advanced Features

### Fingerprint Confidence

Confidence score (0-1) indicates how unique and stable the fingerprint is:

- **0.99+**: Extremely high confidence (ideal)
- **0.95-0.99**: High confidence (good)
- **0.90-0.95**: Medium confidence (acceptable)
- **< 0.90**: Low confidence (flagged)

Low confidence may indicate:
- Privacy browsers (Brave, Tor)
- Missing browser APIs
- Fingerprint spoofing tools
- Very common device configuration

### Incognito Detection

Vouch can detect private/incognito browsing modes:

```json
{
  "deviceFingerprint": {
    "isIncognito": true,
    "incognitoConfidence": 0.92,
    "incognitoSignals": [
      "FILE_SYSTEM_DISABLED",
      "STORAGE_QUOTA_LOW",
      "MISSING_CACHE"
    ]
  }
}
```

### Bot Detection

Identifies automated browsers and scripts:

```json
{
  "deviceFingerprint": {
    "isBotLikely": true,
    "botConfidence": 0.88,
    "botSignals": [
      "NO_MOUSE_MOVEMENT",
      "HEADLESS_BROWSER",
      "SELENIUM_DETECTED",
      "PERFECT_TIMING"
    ]
  }
}
```

## Privacy Considerations

<AccordionGroup>
  <Accordion title="GDPR Compliance" icon="scale-balanced">
    Device fingerprints are hashed and anonymized. They cannot be reverse-engineered to identify a specific person without correlation to other data.

    **Best practices:**
    - Disclose fingerprinting in privacy policy
    - Allow users to request deletion
    - Don't share fingerprints with third parties
  </Accordion>

  <Accordion title="User Transparency" icon="eye">
    Consider informing users about device tracking:

    ```jsx
    <p className="text-sm text-gray-600">
      We use device fingerprinting to prevent fraud and abuse.
      Learn more in our <Link to="/privacy">Privacy Policy</Link>.
    </p>
    ```
  </Accordion>

  <Accordion title="Data Retention" icon="clock">
    Configure how long device fingerprints are stored:

    - **30 days**: Short-term fraud prevention
    - **90 days**: Medium-term patterns
    - **1 year**: Long-term analysis (default)
    - **Forever**: Maximum historical data
  </Accordion>
</AccordionGroup>

## Integration Examples

### React Hook

```jsx
import { useValidateEmail } from '@vouch/react';

function SignupForm() {
  const { validate, result } = useValidateEmail();

  const handleSubmit = async (email) => {
    await validate(email);

    const device = result.signals.deviceFingerprint;

    if (device.previousEmails > 5) {
      // Require additional verification
      setRequirePhoneVerification(true);
    }

    if (device.isBotLikely) {
      // Show CAPTCHA
      setCaptchaRequired(true);
    }

    if (device.riskFactors?.includes('RAPID_ACCOUNT_CREATION')) {
      // Block immediately
      setError('Too many accounts created. Please contact support.');
      return;
    }
  };
}
```

### Server-Side

```javascript
// Server-side API route
app.post('/api/signup', async (req, res) => {
  const { email, fingerprint } = req.body;

  const result = await vouch.validate(email, {
    fingerprint: fingerprint
  });

  const device = result.signals.deviceFingerprint;

  // Log suspicious activity
  if (device.previousEmails > 10) {
    await analytics.track('suspicious_device', {
      deviceId: device.deviceId,
      previousEmails: device.previousEmails,
      riskFactors: device.riskFactors
    });
  }

  // Apply rate limiting
  if (device.emailVelocity === 'very_high') {
    return res.status(429).json({
      error: 'Too many requests from this device'
    });
  }
});
```

## Best Practices

<AccordionGroup>
  <Accordion title="Always Use Client SDKs" icon="code">
    Client SDKs automatically collect accurate fingerprints. Don't try to implement fingerprinting manually.
  </Accordion>

  <Accordion title="Combine with Other Signals" icon="puzzle-piece">
    Device fingerprinting is most powerful when combined with IP reputation, email validation, and behavioral signals.
  </Accordion>

  <Accordion title="Set Appropriate Thresholds" icon="sliders">
    Start with permissive thresholds and tighten based on your fraud patterns. Different applications have different normal usage patterns.
  </Accordion>

  <Accordion title="Don't Block Incognito Users" icon="eye-slash">
    Many legitimate users browse in incognito mode. Flag for review instead of auto-blocking.
  </Accordion>

  <Accordion title="Review False Positives" icon="magnifying-glass">
    Regularly review flagged devices to tune your thresholds and avoid blocking legitimate users.
  </Accordion>
</AccordionGroup>
