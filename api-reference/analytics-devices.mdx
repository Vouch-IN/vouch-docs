## Overview

The `/analytics/devices` endpoint provides insights into device fingerprints collected during email validations. Track unique visitors, device types, and fingerprint confidence scores.

<Note>
  You can use either the versioned endpoint `/v1/analytics/devices` or the latest endpoint `/analytics/devices`
</Note>

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token with your server API key

  ```
  Authorization: Bearer your_server_api_key_here
  ```
</ParamField>

<ParamField header="X-Project-Id" type="string" required>
  Your project ID

  ```
  X-Project-Id: your_project_id
  ```
</ParamField>

<Warning>
  This endpoint requires a **server API key**. Do not use client API keys for analytics endpoints.
</Warning>

## Query Parameters

<ParamField query="startDate" type="string">
  Start date in ISO 8601 format (defaults to 30 days ago)

  ```
  ?startDate=2024-01-01T00:00:00Z
  ```
</ParamField>

<ParamField query="endDate" type="string">
  End date in ISO 8601 format (defaults to now)

  ```
  ?endDate=2024-01-31T23:59:59Z
  ```
</ParamField>

<ParamField query="groupBy" type="string">
  Group results by time interval: `hour`, `day`, `week`, or `month` (defaults to `day`)

  ```
  ?groupBy=day
  ```
</ParamField>

<ParamField query="limit" type="number">
  Maximum number of results to return (defaults to 100, max 1000)

  ```
  ?limit=100
  ```
</ParamField>

<ParamField query="offset" type="number">
  Number of results to skip for pagination (defaults to 0)

  ```
  ?offset=0
  ```
</ParamField>

## Example Request

<CodeGroup>
```bash cURL
curl -X GET "https://api.vouch.expert/analytics/devices?startDate=2024-01-01&endDate=2024-01-31&groupBy=day" \
  -H "Authorization: Bearer your_server_api_key" \
  -H "X-Project-Id: your_project_id"
```

```javascript JavaScript
const response = await fetch(
  'https://api.vouch.expert/analytics/devices?startDate=2024-01-01&endDate=2024-01-31&groupBy=day',
  {
    method: 'GET',
    headers: {
      'Authorization': 'Bearer your_server_api_key',
      'X-Project-Id': 'your_project_id',
    },
  }
);

const data = await response.json();
console.log(data);
```

```python Python
import requests

response = requests.get(
    'https://api.vouch.expert/analytics/devices',
    headers={
        'Authorization': 'Bearer your_server_api_key',
        'X-Project-Id': 'your_project_id',
    },
    params={
        'startDate': '2024-01-01',
        'endDate': '2024-01-31',
        'groupBy': 'day',
    }
)

data = response.json()
print(data)
```
</CodeGroup>

## Use Cases

### 1. Device Distribution Dashboard

Monitor which devices your users are using:

```javascript
const response = await fetch(
  'https://api.vouch.expert/analytics/devices?groupBy=day',
  {
    headers: {
      'Authorization': 'Bearer your_server_api_key',
      'X-Project-Id': 'your_project_id',
    },
  }
);

const { data } = await response.json();

// Visualize device distribution
data.devices.forEach(device => {
  console.log(`${device.platform}: ${device.percentage}%`);
});
```

### 2. Unique Visitor Tracking

Track unique vs. returning visitors:

```javascript
const { data } = await response.json();

const uniqueRate = (data.summary.uniqueVisitors / data.summary.totalValidations) * 100;
const returningRate = (data.summary.returningVisitors / data.summary.totalValidations) * 100;

console.log(`Unique visitors: ${uniqueRate.toFixed(1)}%`);
console.log(`Returning visitors: ${returningRate.toFixed(1)}%`);
```

### 3. Fingerprint Quality Monitoring

Monitor fingerprint confidence scores:

```javascript
const { data } = await response.json();

if (data.summary.averageConfidence < 0.7) {
  console.warn('Low average fingerprint confidence detected');
  // Alert your team or investigate
}
```

## Rate Limits

- **100 requests per minute** per project
- Rate limit headers included in response:
  - `X-RateLimit-Limit`: Maximum requests per window
  - `X-RateLimit-Remaining`: Remaining requests in current window
  - `X-RateLimit-Reset`: Unix timestamp when the window resets

## Best Practices

1. **Cache Results**: Analytics data doesn't change frequently. Cache responses for at least 5-10 minutes.

2. **Use Appropriate Time Ranges**: Request only the data you need to reduce response size and processing time.

3. **Implement Pagination**: For large date ranges, use pagination to avoid timeouts.

4. **Monitor Trends**: Set up alerts for unusual patterns (e.g., sudden drop in unique visitors).

5. **Server-Side Only**: Never expose your server API key to the client. Analytics endpoints should only be called from your backend.
