## Overview

The `/validate` endpoint is the core of Vouch's API. It performs comprehensive email validation including syntax checking, disposable email detection, MX verification, and device fingerprinting.

<Note>
  You can use either the versioned endpoint `/v1/validate` or the latest endpoint `/validate`
</Note>

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token with your API key

  ```
  Authorization: Bearer your_api_key_here
  ```
</ParamField>

<ParamField header="X-Project-Id" type="string" required>

  ```
  X-Project-Id: your_project_id
  ```
</ParamField>

## Request Body

<ParamField body="email" type="string" required>
  The email address to validate

  ```json
  "user@example.com"
  ```
</ParamField>

<ParamField body="fingerprint" type="object">
  Device fingerprint data (automatically included by client SDKs)

  ```json
  {
    "visitorId": "abc123...",
    "confidence": 0.95,
    "signals": { ... }
  }
  ```
</ParamField>

<ParamField body="ip" type="string">
  **Server-side only:** Override IP address for validation

  ```json
  "192.168.1.1"
  ```
</ParamField>

<ParamField body="userAgent" type="string">
  **Server-side only:** Override User-Agent for validation

  ```json
  "Mozilla/5.0 ..."
  ```
</ParamField>

<ParamField body="validations" type="object">
  Override which validations to run (uses project defaults if not specified)

  ```json
  {
    "syntax": true,
    "disposable": true,
    "mx": true,
    "smtp": false,
    "catchall": true,
    "roleEmail": true,
    "alias": true,
    "deviceFingerprint": true,
    "ipReputation": true
  }
  ```
</ParamField>

## Response

<ResponseField name="success" type="boolean" required>
  Whether the validation completed successfully
</ResponseField>

<ResponseField name="valid" type="boolean" required>
  Overall validation result (true if email passed all enabled checks)
</ResponseField>

<ResponseField name="email" type="string" required>
  The original email address submitted
</ResponseField>

<ResponseField name="normalized" type="string" required>
  The normalized/lowercase version of the email
</ResponseField>

<ResponseField name="recommendation" type="string" required>
  Final recommendation: `allow`, `flag`, or `block`

  - **allow**: Email passed all checks, safe to proceed
  - **flag**: Email has suspicious signals, review recommended
  - **block**: Email failed critical checks, should be rejected
</ResponseField>

## Example Request

<CodeGroup>
```bash cURL
curl -X POST https://api.vouch.expert/validate \
  -H "Authorization: Bearer your_api_key" \
  -H "X-Project-Id: your_project_id" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "fingerprint": {
      "visitorId": "abc123",
      "confidence": 0.95
    }
  }'
```

```javascript JavaScript
const response = await fetch('https://api.vouch.expert/validate', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer your_api_key',
    'X-Project-Id': 'your_project_id',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    email: 'user@example.com',
    fingerprint: {
      visitorId: 'abc123',
      confidence: 0.95
    }
  })
});

const result = await response.json();
```

```python Python
import requests

response = requests.post(
    'https://api.vouch.expert/validate',
    headers={
        'Authorization': 'Bearer your_api_key',
        'X-Project-Id': 'your_project_id',
        'Content-Type': 'application/json'
    },
    json={
        'email': 'user@example.com',
        'fingerprint': {
            'visitorId': 'abc123',
            'confidence': 0.95
        }
    }
)

result = response.json()
```

```go Go
package main

import (
    "bytes"
    "encoding/json"
    "net/http"
)

func main() {
    payload := map[string]interface{}{
        "email": "user@example.com",
        "fingerprint": map[string]interface{}{
            "visitorId": "abc123",
            "confidence": 0.95,
        },
    }

    jsonData, _ := json.Marshal(payload)

    req, _ := http.NewRequest("POST", "https://api.vouch.expert/validate", bytes.NewBuffer(jsonData))
    req.Header.Set("Authorization", "Bearer your_api_key")
    req.Header.Set("X-Project-Id", "your_project_id")
    req.Header.Set("Content-Type", "application/json")

    client := &http.Client{}
    resp, _ := client.Do(req)
    defer resp.Body.Close()
}
```
</CodeGroup>

## Rate Limits

| Key Type | Rate Limit | Window |
|----------|------------|--------|
| Client Key | 1,000 requests | Per hour |
| Server Key | 5,000 requests | Per hour |

<Warning>
  Rate limits are per project. If you exceed the limit, you'll receive a 429 error with a `retryAfter` value in seconds.
</Warning>

## Best Practices

<AccordionGroup>
  <Accordion title="Use the Right API Key" icon="key">
    - **Client keys** for browser/mobile apps (includes automatic fingerprinting)
    - **Server keys** for backend validation (can override IP/User-Agent)
    - Never expose server keys in client-side code
  </Accordion>

  <Accordion title="Handle Recommendations Appropriately" icon="shield">
    - **block**: Reject the signup immediately
    - **flag**: Allow signup but mark for review, or require additional verification
    - **allow**: Proceed normally
  </Accordion>

  <Accordion title="Cache Results When Appropriate" icon="database">
    - Cache validation results for repeated validations of the same email
    - Set TTL based on your use case (e.g., 24 hours)
    - Invalidate cache if user changes email
  </Accordion>

  <Accordion title="Provide Good User Experience" icon="user">
    - Show specific error messages (e.g., "Please use a permanent email address")
    - Don't expose internal validation details to users
    - Consider allowing flagged emails with additional verification
  </Accordion>

  <Accordion title="Monitor Your Quota" icon="chart-line">
    - Set up alerts when approaching quota limits
    - Upgrade plan before hitting limits
    - Use test keys during development
  </Accordion>
</AccordionGroup>
